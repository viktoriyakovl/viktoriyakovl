# Glossary

## 1. Введение
Данный документ описывает цели и задачи проекта MIG, определяет границы проекта.

## 2. Перечень сервисов

| Сервис | Описание |
| :--- | :--- |
| VAK | Система работы с альтернативными SMS способами доставки уведомлений на мобильные телефоны |
| SYRI | Сервис по авторизации/ аутентификации и SSO |
| SYRR | Сервис по регистрации новых пользователей |
| SYRP | Сервис по управлению аккаунтом (смена пароля, имени, удаление игровой сессии, привязка социальных сетей и т.п.). Участвует в восстановлении аккаунта |
| SYRC | Сервис по доставке уведомлений (внутриигровые уведомления, email, sms и т.д.) |
| BB | Сервис для получения данных об аккаунте и управления этим аккаунтом |
| MailPost | Вендор мобильного маркетинга и коммуникаций |

## 3. Термины и определения

| Термин | Обозначение | Расшифровка | Определение |
| :--- | :--- | :--- | :--- |
| Альтернативный сценарий | n-A | Alternative flow | Отклонение от стандартного сценария, которое в итоге приводит к ожидаемому результату |
| Асинхронная передача данных | - | - | Способ передачи данных, при котором каждый блок данных обрамляется стартовым и стоповым сигналами, а взаимодействующие устройства не используют общий тактовый сигнал, обмениваясь данными в произвольные моменты времени |
| Аутентификационное сообщение | - | - | Служебное сообщение, которое подтверждает права пользователя на доступ к данным или функциям системы |
| Бизнес-правило | BR | Business Rules | Политика, предписание, стандарт, правило или вычислительная формула, определяющая или ограничивающая некоторые стороны бизнес-процессов |
| Бизнес-риск | BRisk | Business Risk | Потенциальная угроза того, что проект не достигнет своих коммерческих и стратегических целей, что приведет к финансовым потерям, ущербу для репутации или упущенным выгодам для компании |
| Бизнес-требование | - | - | Объем информации, который в совокупности описывает потребность, которая инициирует один или больше проектов, призванных предоставить решение и получить требуемый конечный бизнес-результат. Бизнес-требования включают бизнес-возможности, бизнес-цели, метрики успеха, концепция и границы, а также ограничения |
| Бизнес-цель | BO | Business Objective | Финансовая или не финансовая выгода, которую организация ожидает получить в результате реализации проекта или другой инициативы |
| Брокер сообщений | - | - | Программное обеспечение для связи между приложениями, системами и службами, помогающее им обмениваться информацией друг с другом. Это делается посредством перевода сообщений из одного формального протокола обмена сообщениями в другой |
| Вендор | - | - | Компания или независимый поставщик, который продает товары или услуги другим компаниям или конечным потребителям |
| Дашборд | - | - | Набор метрик по параметрам в графическом представлении для удобного наблюдения |
| Допущение | A | Assumption | Допущение, на котором основываются границы проекта |
| Зависимость | DE | Dependency | Зависимость успешного выполнения проекта от других сервисов, проектов или команд |
| Идентификатор | ID | Identifier | Постоянный уникальный идентификатор |
| Интеграционное требование | IR | Integration Requirement | Описание того, как разрабатываемая система должна взаимодействовать с внешними системами, компонентами, устройствами и сервисами |
| Интеграция | INT | Integration | Взаимодействие разрабатываемого сервиса с другими сервисами |
| Исключение | n-E | Exception | Отклонение от стандартного сценария, которое не приводит к ожидаемому постусловию |
| Ключевая функция | MF | Major Feature | Высокоуровневая фундаментальная возможность, которую система должна иметь, чтобы реализовать основное видение продукта и принести ценность для бизнеса и пользователя |
| Контейнер | - | - | Легковесный, изолированный исполняемый пакет программного обеспечения, который содержит все необходимое для запуска приложения: код, среду выполнения, системные инструменты, библиотеки и настройки |
| Контекстная диаграмма | - | - | Аналитическая модель, которая описывает абстрактную систему высокого уровня. Контекстная диаграмма определяет внешние для системы объекты, которые взаимодействуют с ней, но не отображает ничего из внутренней структуры или поведения системы |
| Лог | - | - | Данные, генерируемые сервисом или продуктом с информацией о том, что было сделано в системе |
| Маркетинговое сообщение | - | - | Рекламное сообщение, которое информирует о продукте, акции или новости с целью привлечения клиентов и стимуляции продаж |
| Метрика | - | - | Качественные или количественные показатели наблюдаемой системы в дискретный момент времени |
| Нефункциональное требование | NF | Nonfunctional Requirement | Описание присущих свойств или характеристик, которые система должна демонстрировать, или ограничения, которые она должна соблюдать |
| Ограничение и исключение | LE | Limitation and Exclusion | Возможность или характеристика, которые могут ожидаться заинтересованными в проекте лицами, но включение которых в продукт или в его определенную версию не запланировано |
| Ограничение проектирования | C | Constraint | Ограничение, налагаемое на архитекторов и разработчиков системы |
| Персональные данные | ПД | Персональные данные | Любая информация, позволяющая прямо (например, номер телефона) или косвенно (например, IP-адрес) идентифицировать физическое лицо |
| Программный интерфейс приложения | API | Application Programming Interface | Контракт (условно «договор»), который предоставляет программа. Формальное обязательство связывать между собой различные программируемые контексты.<br>API включает:<br>• операцию, которую возможно выполнить посредством данного API;<br>• входящие данные;<br>• исходящие данные – контент данных или сообщение об ошибке |
| Республика Беларусь | РБ | Республика Беларусь | - |
| Российская Федерация | РФ | Российская Федерация | - |
| Синхронная передача данных | - | - | Способ передачи данных, при котором взаимодействующие устройства синхронизированы по общему тактовому сигналу, что позволяет передавать данные в виде непрерывного потока без дополнительных служебных битов на каждый символ |
| Стандартный сценарий | - | - | Сценарий, в рамках которого самым быстрым способом достигается ожидаемый результат |
| Таймаут | Timeout | - | Максимальный промежуток времени, по истечении которого сервис прерывает операцию и считает ее неудачной |
| Федеральный закон N 223 | ФЗ-223 | - | Федеральный закон "О закупках товаров, работ, услуг отдельными видами юридических лиц" от 18.07.2011 N 223-ФЗ (последняя редакция) |
| Функциональное требование | FR | Functional Requirements | Описание поведения системы в определенных условиях |
| Функция обратного вызова | Callback | - | Механизм, при котором один сервис передает другому адрес для возврата данных о результате выполнения задачи, когда она будет завершена |
| CI/CD | CI/CD | Continuous Integration / Continuous Delivery (Deployment) | Комбинация непрерывной интеграции и непрерывного развертывания программного обеспечения в процессе разработки |
| Docker | - | - | Платформа для разработки, доставки и запуска приложений в изолированных, легковесных контейнерах |
| Elasticsearch | - | - | Тиражируемая программная поисковая система. Горизонтально масштабируемая, позволяет быстро в режиме реального времени хранить, искать и анализировать большие объемы данных |
| Flash Call | - | - | "Звонок с прокидкой номера" - технология автоматической идентификации, используемая для подтверждения действий (технология передачи кода подтверждения посредством указания нескольких последних цифр номера телефона, с которого осуществляется вызов пользователю) |

Поддерживает основные технологии контейнеризации, включая Docker, также возможна поддержка технологий аппаратной виртуализации |

| rps | rps | Request per second | Количество запросов в секунду |
| Sentry | - | - | Платформа отслеживания ошибок, помогающая разработчикам диагностировать, исправлять и оптимизировать производительность своего кода |

| User Story | US | User Story | Описание по модели потребностей и действующих лиц.<br>Способ выражения пользовательских требований в проектах гибкой разработки в форме одного или двух предложений, формулирующих потребность пользователя или описывающих единицу необходимой функциональности, а также говорящих о пользе, какую эта функциональность приносит пользователю |
| Vault | Vault | - | Специализированное приложение для централизованного и безопасного управления конфиденциальными данными с контролем доступа |
| Wallarm | Wallarm | - | Брандмауэр веб-приложений, предназначенный для защиты сервисов и API |
